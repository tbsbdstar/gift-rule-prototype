1. 直接用浏览器打开 `玛鲁丸_赠品规则系统_v1.5.html` 即可（纯前端，无需后端）。

### 3.1 统计卡片（顶部 4 张）
- 规则总数、启用中、满金额赠、满件数赠（随数据实时统计）。

### 3.2 筛选与查询（主动查询）
- 筛选项：玩法类型、贸易类型、状态（全部 / 启用中 / 已停用）、规则名称。
- **主动查询**：下拉改变后不会自动刷新，需点击「🔍 查询」才生效；规则名称输入框支持回车查询。
- 「重置」一键清空全部筛选条件并恢复全部列表。

### 3.3 规则表格列
| 列 | 内容 |
|---|---|
| 规则名称 | 名称 + 副标题「付款锁定」 |
| 玩法 | 满金额赠 / 满件数赠（标签） |
| 贸易类型 | 一般贸易 / 海外直邮 / 保税仓（标签） |
| 门槛区间 | 金额：`¥X 起` 或 `¥X ~ ¥Y`；件数：`X件 起` 或 `X ~ Y件` |
| 商品范围 | **全部商品 / 指定商品 / 除指定外 / 分类商品**（仅显示方式标签，不显示编码与数量） |
| 赠品配置 | **随机发 / 单种发**（仅显示模式，不显示几选几或编码） |
| 优先级 | 数字 |
| 有效期 | 起止日期；长期规则显示「长期」标签 |
| 状态 | 开关（即时启用/停用） |
| 操作 | 详情 / 编辑 / 复制 / 删除 |

### 3.4 行操作
- **详情**：弹窗只读查看完整配置。
- **编辑**：进入整页向导，回填该规则全部配置。
- **复制**：生成「（副本）」，默认停用，供修改后启用。
- **删除**：二次确认后删除。
- **状态开关**：直接切换启用/停用。

## 四、新建 / 编辑规则（整页式 4 步向导）

整页呈现（非弹窗）：保留侧边栏与顶栏，顶部为 4 步进度条，**每步独占一页**，底部固定操作栏（按钮成组靠右：`取消 ｜ 上一步 ｜ 下一步`，末步为 `✓ 保存规则`）。点「下一步」校验当前步，校验不过停留并提示「请先完善必填项」。

### 第 1 步：基础信息与门槛
- **规则名称**（必填）
- **① 贸易类型（单选）**：一般贸易 / 海外直邮 / 保税仓。决定子单生效范围与赠品池；切换贸易类型会清空已选的商品范围编码与赠品（二次确认）。
- **② 玩法类型**：满金额赠 / 满件数赠（影响门槛单位与文案：金额↔件数）。
- **③ 门槛区间**：最低（必填）～ 最高（选填，空 = 无上限）；最高需 ≥ 最低。
- **④ 有效期**：起止日期；勾选「长期」后日期框置灰，表示任何时间有效。
- **⑤ 优先级**：数字越小越优先。
  - 释义：当**相同玩法类型且相同贸易类型**的规则有效期重叠时，只触发其中一条——取「启用中、下单时在有效期内、优先级数字最小」的那条，其余忽略；不同玩法或不同贸易类型互不影响、可同时触发。
- **锁定时机说明**：统一为「付款成功时锁定」（普通订单付款、预售付定金均按付款节点，不区分订单类型）。

### 第 2 步：商品参与范围
顶部提示：在所选贸易类型的商品池内圈定计入门槛的商品；**组合商品不参与**（既不计入门槛，也不能作赠品）。

四种方式：
1. **该贸易类型全部商品**：该贸易类型下所有商品均参与门槛计算。
2. **指定具体商品（规格编码）**：输入/导入规格编码，查询结果表展示。
3. **指定商品以外（排除）**：输入需排除的规格编码，其余商品参与。
4. **按商品分类**：三级级联下拉（一级 → 二级 → 三级），**只显示分类名称、不显示编号**；可只选大分类（含其全部下级），下方实时显示已选范围（如「动漫周边 · 全部子类」）。

> 切换范围方式会**重置当前方式的已选项**（导航返回同一方式则保留）。

#### 规格编码查询结果表（方式②③）
列：**规格编码 · 商品名称 · 贸易类型 · 售卖方式（现货/预售）· 单价 · 操作**
- 支持多编码同时查询；结果区顶部显示「共 N 个 / 有效 X / 异常 Y」与「清空全部」。
- 列表**固定高度滚动 + 表头吸顶**（不分页），编码多也可顺畅浏览。
- 点行尾「×」删除对应编码（结果同步移除）。
- 异常行标红并拦截：编码未找到 / 贸易类型与规则不一致 / 组合商品 / 编码超长。

### 第 3 步：配置赠品
- **不需要选择赠品来源、不需要选择单种/多种**：
  - 只要编码存在于商品资料即可添加；
  - **填 1 个编码 = 单种固定赠送；填多个 = 多种随机发**，模式由编码数量自动判定。
- 多种随机发提示：满足门槛时从**有库存**的赠品中随机发 1 种；若全部无库存则本次不赠。
- **赠品查询结果表**列：**规格编码 · 商品名称 · 贸易类型 · 在库库存 · 操作**（不含赠送数量）。
- 拦截：组合商品不可作赠品 / 编码未找到 / 贸易类型不一致 / 编码超长。

### 第 4 步：确认提交
- 汇总「基础与门槛 / 商品参与范围 / 赠品配置」三块配置。
- **优先级冲突预检**：当存在**同玩法 + 同贸易类型 + 有效期重叠 + 优先级相同**的规则时，红色报错并**禁用保存**，提示与之冲突的规则名。
- **共存提示（非阻断）**：当存在同玩法 + 同贸易类型 + 有效期重叠但**优先级不同**的规则时，提示「可共存保存，下单时按『启用 + 当时在有效期内 + 优先级最小』自动取一条触发」。
- **全部赠品无库存提示**：达门槛订单本次不赠，库存恢复后新订单恢复赠送。
- 保存成功后关闭向导、回到列表并提示。

## 五、批量导入（第2步 / 第3步共用）

每个编码输入栏旁有「📥 批量导入」按钮，打开导入弹窗：

- **录入方式**：① 上传文件（`.xls / .csv / .txt`）；② 直接在文本域粘贴。
- **分隔符**：支持 逗号(英/中) / 空格 / 换行 / 分号 / 顿号 任意分隔；表格中一行一个编码也可（无需加逗号）。
- **自动规整**：解析时**自动去空格、转大写、去重**；自动忽略 `#` 开头说明行、表头词、含中文的内容（名称/注释行不会被当成编码）。
- **实时计数**：文本域上方显示「已识别 N 个编码」。
- **导入模板下载**：弹窗内「⬇ 下载导入模板」生成 `满赠规则_编码导入模板.xls`：
  - 三行一列、黑色边框：第1行「规格编码」（名称）、第2行「必填，规格编码（13或15位）」（注释）、第3行起为示例/可填编码；下方含可填空行；启用整表网格线。
- **数量 / 长度限制**（手输与导入均生效）：
  - 单个编码 ≤ **32 位**：超长作为**红色异常行报错**（疑似多个编码未分隔或内容有误），并拦截保存。
  - 指定/排除编码 ≤ **200 个**、赠品编码 ≤ **20 个**：超出部分**不予添加**，明确提示未添加数量（不静默截断）。

## 六、校验与拦截规则汇总

| 场景 | 行为 |
|---|---|
| 第1步必填缺失（名称/贸易类型/门槛/有效期） | 标红 + 提示「请先完善必填项」，不放行 |
| 门槛最高 < 最低 | 校验不过 |
| 有效期结束早于开始（未勾长期时） | 校验不过 |
| 指定/排除方式无有效编码 | 校验不过 |
| 按分类未选一级分类 | 校验不过 |
| 赠品无有效项 | 校验不过 |
| 编码未找到 / 贸易类型不一致 / 组合商品 / 超长 | 红色异常行 + 拦截，需移除 |
| 同玩法+同贸易类型+有效期重叠+同优先级 | 冲突，禁用保存 |

## 八、内置模拟数据

- **商品库 `SKU_DB`**：按贸易类型隔离，含字段 `name / trade / source / stock / sale(现货|预售) / price / combo`。示例：`SKU-001`（一般贸易·现货·有货）、`SKU-002`（无库存）、`SKU-088`（海外直邮）、`SKU-055/056`（保税仓）、`GIFT-008`（商品资料）、`SET-001/SET-088`（组合商品，不可参与）。
- **分类树 `CATEGORY_TREE`**：一级「动漫周边」下含 手办雕像 / 模型玩具 / 玩偶娃娃 / TCG卡牌 / 日用周边 / 一番赏盲盒 / 谷子 等二级及其三级（编码内部保存、界面只显示名称）。
- **示例规则 `RULES`**：夏季满赠 A/B/C（对应三种贸易类型）、满10件送惊喜（长期、满件数赠）。

## 九、与说明书（v1.5）的差异（以页面为准）

> 以下为页面在迭代中相对说明书的调整，**实际行为以页面为准**。

1. **取消「订单类型」选择**：说明书区分普通/预售订单的锁定时机；页面**统一为「付款成功锁定」**，不在界面区分订单类型。
2. **取消「赠品来源」选择**：说明书有「已上架商品 / 商品资料」两种来源选择；页面取消该选择，**只要编码存在于商品资料即可添加**。
3. **单种/多种改为自动判定**：说明书需手动选「单种 / 多种随机」；页面**按赠品编码数量自动判定**（1 个=单种固定，多个=随机发）。
4. **赠品不设「赠送数量」**：页面赠品结果表已移除赠送数量列（默认每种 1 件）。
5. **商品范围方式④为「按商品分类」**：说明书为「按分类 / 属性」；页面收敛为**三级商品分类级联**（不含品牌/标签）。
6. **优先级冲突范围更严**：说明书按「同玩法类型」判定；页面收紧为**同玩法 + 同贸易类型**才算竞争/冲突（不同贸易类型作用于各自子单，互不影响）。
7. **新增「长期」有效期**：长期活动任何时间有效，与同类规则仍比较优先级。
8. **新增编码数量/长度限制与批量导入/模板**：说明书未明确，页面新增 32 位长度上限、200/20 数量上限、导入与模板下载。
9. **退款（第7章）、赠品锁定全流程（第6章）、发货（第8章）等运行时流程**：说明书有详细描述；**当前页面聚焦"规则配置闭环"，未实现这些运行时/订单侧界面**（如需可另行扩展）。

## 十、术语速查

- **规格编码（SKU编码）**：商品的唯一规格标识（一般13位/部分15位），仓库按此拣货发货。
- **有效金额**：参与商品原价之和，不含满减/优惠券/运费/组合商品。
- **组合商品**：既不计入门槛，也不能作赠品，输入即被拦截。
- **子单**：平台按贸易类型拆分结算后的独立订单，满赠规则作用于子单。
